# typedsigrokdecode

Try to make the development process of sigrokdecode decoders more IDE-friendly.

> [!IMPORTANT]
> Due to the design of Python typing system, the put() method is typed to accept fixed length tuples, thus it requires a patch to the put() method in libsigrokdecode to allow it to accept tuples as well as lists. Mainline libsigrokdecode will NOT work properly with this library. Effort will be made to push this patch to upstream so hopefully one day this situation will change.

## Design

typedsigrokdecode is designed to be able to be used both externally in a virtual environment, and as an common library that can be shipped with libsigrokdecode.

No newer typing syntaxes, such as the new Generic or Union syntax, are used in typedsigrokdecode, in order to maintain compatibility with older Python versions. However, even so, typedsigrokdecode still requires the target libsigrokdecode to be built with Python 3.8+. 

## Usage

> [!NOTE]
> You should at least be familiar with how an untyped decoder works in order to understand how a typed decoder would work. It is advised that you have a quick read on [Protocol decoder API/Queries](https://sigrok.org/wiki/Protocol_decoder_API/Queries) and [Protocol decoder HOWTO](https://sigrok.org/wiki/Protocol_decoder_HOWTO) in order to have a general understanding of how an untyped decoder is structured. Note that they may contain obsolete information, so take them with a grain of salt and cross check with other decoders to make sure.
>
> (Or just go ahead and experiment with this library. Play with it in an IDE, inspect the `typedsigrokdecode.typing` module, etc. The interface should be pretty easy to navigate.)

The process of defining the decoder is similar to how an untyped sigrokdecode decoder would be defined, with some important differences:

- Instead of basing on the `sigrokdecode.Decoder` abstract class, the Decoder class should be based on the decoder Protocol class from `typedsigrokdecode`. That is, either the `typedsigrokdecode.BottomDecoder[<OutputPythonType>]` class or the `typedsigrokdecode.StackedDecoder[<InputPythonType>, <OutputPythonType>]` class depending on the decoder type.
  - For a non-stacked decoder that has `['logic']` as the input type list, use `typedsigrokdecode.BottomDecoder`.
  - For a stacked decoder that works on outputs generated by other decoders, use `typedsigrokdecode.StackedDecoder`.
  - Replace `<InputPythonType>` with the custom output type exported by lower level decoders, and export custom output type with `<OutputPythonType>`.
- For each optional property or method, the Decoder must also implement the corresponding Protocol class (available as `Has*` and `Supports*` classes under `typedsigrokdecode` or `typedsigrokdecode.typing`) to enable type checking on that property or method.
- At the bottom of the pd module, it is highly recommend to try to instantiate the decoder class inside a `TYPE_CHECKING` block. THis is to allow the type checker to ensure that all abstract properties and methods have been implemented correctly.
- To get configuration options set on the decoder instance in a type-safe way, instead of using `self.options`, the decoder shall use the `self.get_options()` method.

Below is what a `pd.py` file developed with `typedsigrokdecode` would look like:

```python
from typing import TYPE_CHECKING, Final, Literal, Optional, TypeAlias
from typedsigrokdecode import OUTPUT_ANN, SRD_CONF_SAMPLERATE, BottomDecoder, HasAnnotationRows, HasAnnotations, HasChannels, HasOptionalChannels, HasOptions


SCL: Final = 0
SDA: Final = 1


PythonPacket: TypeAlias = None


class Decoder(BottomDecoder[PythonPacket],
              HasOptions,
              HasChannels,
              HasOptionalChannels,
              HasAnnotations,
              HasAnnotationRows,
              SupportsMetadata):
    api_version = 3
    id = 'i2c'
    name = 'I²C'
    longname = 'Inter-Integrated Circuit'
    desc = 'Two-wire, multi-master, serial bus.'
    license = 'gplv2+'
    inputs = ['logic']
    outputs = ['i2c']
    tags = ['Embedded/industrial']
    channels = (
        {'id': 'scl', 'name': 'SCL', 'desc': 'Serial clock line'},
        {'id': 'sda', 'name': 'SDA', 'desc': 'Serial data line'},
    )
    optional_channels = ()
    options = (
        {'id': 'address_format', 'desc': 'Displayed slave address format',
           'default': 'shifted', 'values': ('shifted', 'unshifted')},
    )
    annotations = (
        ('start', 'Start condition'),
        ('repeat-start', 'Repeat start condition'),
        ('stop', 'Stop condition'),
        ('ack', 'ACK'),
        ('nack', 'NACK'),
        ('bit', 'Data/address bit'),
        ('address-read', 'Address read'),
        ('address-write', 'Address write'),
        ('data-read', 'Data read'),
        ('data-write', 'Data write'),
        ('warnings', 'Human-readable warnings'),
    )
    annotation_rows = (
        ('bits', 'Bits', (5,)),
        ('addr-data', 'Address/Data', (0, 1, 2, 3, 4, 6, 7, 8, 9)),
        ('warnings', 'Warnings', (10,)),
    )

    samplerate: Optional[int] = None
    state: Literal['FIND START'] = 'FIND START'  # Add more states here as you need

    def metadata(self, key: int, value: int) -> None:
        if key == SRD_CONF_SAMPLERATE:
            self.samplerate = value

    def reset(self) -> None:
        # Reset inner states 
        ...

    def start(self) -> None:
        # Register channels
        self.out_ann = self.register(OUTPUT_ANN)
        # Parse options
        options = self.get_options()
        # Do something with options

    def decode(self) -> None:
        while True:
            self.wait({SCL: 'f'})
            # Do more things here


if TYPE_CHECKING:
    from typing import reveal_type
    reveal_type(Decoder())
```
